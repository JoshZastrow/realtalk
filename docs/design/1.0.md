# Design Implementation Spec — v1.0

> **Purpose:** This document tells a frontend engineer exactly what to change in the
> realtalk TUI to match the Intimate Manuscript design system defined in `DESIGN.md`.
> It maps every widget and screen to specific before/after changes with code-level detail.

## Current State Audit

The game currently uses Textual defaults with `CSS = ""` in `app.py`. Every screen and
widget is unstyled. Visual findings from audit run on 2026-04-15:

| Element | Current | Target |

## 2. StatusBar Widget

**File:** `realtalk/tui/widgets.py`
**Current:** `realtalk/tui/widgets.py:StatusBar.render_text()` lines ~13-18

### Current implementation

```python
def render_text(self) -> str:
    filled = max(0, min(20, round((self.value / self.max_value) * 20)))
    bar = "[" + ("=" * filled).ljust(20) + "]"
    suffix = f" {self.delta_label}" if self.delta_label else ""
    return f"{self.label} {bar} {self.value}{suffix}"
```

Output: `MOOD [===========         ] 50`

### Target implementation

```python
MOOD_LABELS: dict[range, str] = {
    range(65, 101): "Warm",
    range(35, 65): "Strained",
    range(0, 35): "Fragile",
}

SECURITY_LABELS: dict[range, str] = {
    range(65, 101): "Grounded",
    range(35, 65): "Uneasy",
    range(0, 35): "Exposed",
}

MOOD_COLORS: dict[range, str] = {
    range(65, 101): "#7da85a",
    range(35, 65): "#c4a03a",
    range(0, 35): "#9e4a4a",
}


def _label_for(value: int, table: dict[range, str]) -> str:
    for r, label in table.items():
        if value in r:
            return label
    return ""


def block_bar(value: int, width: int = 10) -> str:
    filled = round(value / 100 * width)
    bar = "█" * filled
    if filled < width:
        bar += "▒"
        bar += "░" * (width - filled - 1)
    return bar


class StatusBar(Static):
    def __init__(
        self,
        label: str,
        value: int,
        max_value: int = 100,
        delta_label: str = "",
        color: str = "green",   # kept for backwards compat, unused after redesign
    ) -> None:
        self.label = label
        self.value = value
        self.max_value = max_value
        self.delta_label = delta_label
        super().__init__(self.render_text())

    def render_text(self) -> str:
        label_table = MOOD_LABELS if self.label == "MOOD" else SECURITY_LABELS
        state_label = _label_for(self.value, label_table)
        bar = block_bar(self.value)
        return f"{self.label}  {bar}  {state_label}  {self.value}"

    def refresh_bar(self, value: int, delta_label: str = "") -> None:
        self.value = value
        self.delta_label = delta_label
        self.update(self.render_text())
```

Output (mood=84): `MOOD  ████████▒░  Warm  84`
Output (mood=38): `MOOD  ███▒░░░░░░  Strained  38`
Output (mood=14): `MOOD  █░░░░░░░░░  Fragile  14`

### Bar color

The status bar color should change with value tier. Add a `color` property to the
`StatusBar` widget that returns the correct hex for the current value, then apply it
via Rich markup or CSS:

```python
@property
def bar_color(self) -> str:
    for r, color in MOOD_COLORS.items():
        if self.value in r:
            return color
    return "#7da85a"

def render_text(self) -> str:
    label_table = MOOD_LABELS if self.label == "MOOD" else SECURITY_LABELS
    state_label = _label_for(self.value, label_table)
    bar = block_bar(self.value)
    color = self.bar_color
    return f"{self.label}  [{color}]{bar}[/{color}]  {state_label}  {self.value}"
```

## 3. Mood-Tier Accent Color

**File:** `realtalk/tui/screens.py` — `GameScreen`

The accent color (`--accent` CSS variable) must update when mood crosses a tier
boundary. Toggle a CSS class on the screen root.

```python
# In GameScreen, after updating mood value:
def _update_mood_class(self) -> None:
    mood = self.mood.value
    self.remove_class("mood-warm", "mood-mid", "mood-cold")
    if mood >= 65:
        self.add_class("mood-warm")
    elif mood >= 35:
        self.add_class("mood-mid")
    else:
        self.add_class("mood-cold")
```

Call `_update_mood_class()` in `on_mount()` (initial state) and inside `_run_step()`
after `self.mood.refresh_bar(...)`.

Initial class should be set in `on_mount` based on starting mood value.

## 4. ASCII Scene Headers

**File:** `realtalk/tui/screens.py` — `SituationScreen.compose()` and `GameScreen.compose()`

Add an ASCII art header above the dialogue. Each scene type gets one illustration.

### ASCII art constants

Add to `realtalk/tui/screens.py` (or a new `realtalk/tui/ascii_art.py`):

```python
SCENE_HEADERS: dict[str, str] = {
    "coffee_shop": """\
   ( )
  (   )
   \\_/
  ──────""",
    "late_night_office": """\
    💡
  ┌───┐
  │   │
  └───┘""",
    "hiking_trail": """\
    /\\
   /  \\
  /────\\
  ────""",
    # fallback for any scene not listed
    "default": """\
  ·  ·  ·
 · · · · ·
  ·  ·  ·""",
}
```

For scenes not in this dict, use `"default"`. The scene ID comes from
`self.engine.current_state().scene` (check the exact field name in `session.py`).

### Header widget

```python
class SceneHeader(Static):
    def __init__(self, scene_id: str, scene_name: str, role_name: str) -> None:
        art = SCENE_HEADERS.get(scene_id.lower().replace(" ", "_"), SCENE_HEADERS["default"])
        label = f"{scene_name.upper()} · {role_name.upper()}"
        separator = "─" * 40
        content = f"[dim]{art}[/dim]\n[dim]{label}[/dim]\n[dim]{separator}[/dim]"
        super().__init__(content)
```

### Add to SituationScreen and GameScreen

In `SituationScreen.compose()`, yield the header before the body:
```python
def compose(self) -> ComposeResult:
    state = self.engine.current_state()
    yield SceneHeader(state.scene, state.scene_name, state.role_name)
    yield self._body
```

In `GameScreen.compose()`, yield the header at the top:
```python
def compose(self) -> ComposeResult:
    state = self.engine.current_state()
    yield SceneHeader(state.scene, state.scene_name, state.role_name)
    yield self.dialogue
    yield Static("", classes="separator")
    yield self.mood
    yield self.security
    yield Static("", classes="separator")
    yield self.reaction
    yield Static("", classes="separator")
    yield self.options
    yield self._status
```

Check `session.py` for the exact attribute names on `GameState` (likely `scene_id`,
`scene_name`, `role_name`, or similar).

## 6. Dialogue Text Styling

**File:** `realtalk/tui/widgets.py` — `DialogueArea`

The dialogue area should render NPC dialogue in bold, scene descriptions in italic dim.

The current `DialogueArea.append_text()` appends raw text. The game engine can signal
whether text is dialogue vs. description by convention (quotes = dialogue, no quotes =
narration), or `GameScreen._run_step()` can wrap the LLM response in Rich markup before
calling `append_text`.

Minimum change: apply `[bold]` to all dialogue text for now.

```python
# In GameScreen._run_step(), when updating dialogue:
self.dialogue.append_text(f"[bold]{new_text}[/bold]")
```

Full implementation: parse the response to detect dialogue lines (quoted strings) vs.
narration lines (unquoted), apply `[bold]` and `[italic dim]` respectively.

## 8. ReactionInput Display

**File:** `realtalk/tui/widgets.py` — `ReactionInput._render_text()`

The reaction input already shows live state. Style the active reaction with the
accent color when valid:

```python
def _render_text(self) -> str:
    if not self.raw_value:
        return "[dim]Reaction: type a/r + 1-3  (e.g. a2 = attract medium, r1 = repel light)[/dim]"
    if self.is_valid:
        direction_label = "attract" if self.direction == "a" else "repel"
        return f"Reaction: [$accent]{self.raw_value}[/$accent]  ({direction_label}, intensity {self.intensity}) — press 1/2/3 to respond"
    return f"[dim]Reaction: {self.raw_value}_  (a=attract, r=repel, then 1-3)[/dim]"
```

Note: `$accent` is a CSS variable, not directly usable in Rich markup. Options:
- Read the accent color from the parent widget's CSS and pass it in, OR
- Use a fixed color that maps to the current tier (simpler: just use `#c4703a` as default)

Simplest working version: use the mid-tier amber as fixed accent for the valid state,
which can be refined once dynamic accent is wired up.

## 9. Menu Screens (Scene + Role Pickers)

**Files:** `realtalk/tui/widgets.py` — `MenuList`; `realtalk/tui/screens.py` — `SceneScreen`, `RoleScreen`

### MenuList styling

`MenuList` currently renders items as `> 1. Item` and `  2. Item` (with `>` prefix and
spaces). Style the selected item in accent color, dim the unselected items:

```python
def render_text(self) -> str:
    lines = []
    for i, item in enumerate(self.items):
        if i == self.selected_index:
            lines.append(f"[bold][$accent]→ {i+1}. {item}[/$accent][/bold]")
        else:
            lines.append(f"[dim]  {i+1}. {item}[/dim]")
    return "\n".join(lines)
```

Again, `$accent` not directly usable in Rich markup — use `#c4a03a` (mid-tier amber)
as a fixed color until dynamic accent is wired. The visual effect (bright selected vs.
dim unselected) is the important part.

Mono via CSS — the user's terminal must be configured to use it.

## 12. Animation (Optional / Phase 2)

These are marked as optional but are called out in DESIGN.md:

| Animation | How |
|-----------|-----|
| Dialogue streaming at ~20ms/char | `GameScreen._run_step()` — stream response token-by-token using `asyncio.sleep(0.02)` between `append_text()` calls |
| Status bar ease on change | `StatusBar.refresh_bar()` — use Textual's `self.animate("value", new_value, duration=0.3)` after setting up `value` as a reactive attribute |
| Accent color crossfade | CSS transition — add `transition: --accent 300ms;` if Textual supports CSS transitions (check version) |

## Implementation Order

Do these in order. Each step is independently shippable.

1. **Step 1 — Light mode base**: Set `CSS` in `app.py` with background `#f5f0e8`, text `#2a1f15`. This single change flips the game from dark to light and has the biggest visual impact.
2. **Step 2 — StatusBar redesign**: Rewrite `StatusBar.render_text()` to use block characters and emotional labels. No CSS needed.
3. **Step 3 — Action zone separation**: Add `.action-zone` class + CSS, add separator lines in `GameScreen.compose()`.
4. **Step 4 — Mood tier classes**: Add `_update_mood_class()` to `GameScreen`, wire it to `on_mount` and `_run_step`.
5. **Step 5 — ASCII headers**: Add `SCENE_HEADERS` dict and `SceneHeader` widget, yield it in `SituationScreen` and `GameScreen`.
6. **Step 6 — Options + dialogue styling**: Dim option prefixes, bold dialogue text.
7. **Step 7 — Title bar**: Add `#title-bar` to `GameScreen`.
8. **Step 8 — Animations** (Phase 2): Streaming text, status bar easing.

## Acceptance Criteria

A passing implementation produces a TUI where:

- [ ] Background is parchment `#f5f0e8`, not dark
- [ ] Status bars show `MOOD  ████████▒░  Warm  84` format
- [ ] MOOD label reads Warm / Strained / Fragile based on value range
- [ ] SECURITY label reads Grounded / Uneasy / Exposed based on value range
- [ ] Status bar fill characters match value (10-char wide)
- [ ] Status bar color is green for high, amber for mid, red for low
- [ ] Accent color shifts with mood tier on every game step
- [ ] ASCII scene header appears at top of SituationScreen and GameScreen
- [ ] Dialogue zone has no border; text is padded
- [ ] Action zone (mood bars, reaction, options) has a visible top separator
- [ ] Option numbers are dim; option text is normal weight
- [ ] Title bar shows `realtalk` left, `^C quit` right in muted color
- [ ] Menu picker shows selected item highlighted, others dim
